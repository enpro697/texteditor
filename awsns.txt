### 正解の導き出し (Unveiling the Correct Answer)

---

(Io) Okay, so VIP S3 cards don't work, and playing whack-a-mole with IP addresses will ruin my chill. So how do we actually force the ALB to only talk to CloudFront? Tell me there's a smarter way that doesn't involve me doing math or memorizing lists.

(Io) オッケー、つまりS3用のVIPカードは使えないし、IPアドレスでモグラ叩きするのは私のチルな時間をぶち壊すってことね。じゃあ、実際どうやってALBにCloudFrontとだけ会話させるの？ 計算したりリスト暗記したりしなくていい、もっとスマートな方法があるって言ってよ。

(Rindo) There is indeed a highly elegant and standard solution. The correct answer is to configure CloudFront to add a custom HTTP header to the requests it forwards, and then configure an ALB listener rule to validate that specific custom header.

(Rindo) 極めて優雅で、かつ標準的な解決策が確かに存在する。正解は、転送するリクエストにカスタムHTTPヘッダーを追加するようにCloudFrontを設定し、ALBのリスナールールでその特定のカスタムヘッダーを検証するように設定することだ。

(Io) Custom... HTTP header? Wait, I need a translation for that. Is that like a secret handshake?

(Io) カスタム…HTTPヘッダー？ 待って、それ翻訳が必要かも。それって秘密の握手みたいなもの？

(Rea) A secret handshake is a surprisingly apt analogy for your usually vacant mind, Iochi. Yes. You inject a secret passcode—a custom header like `X-My-Secret-Header: BloodOath123`—into CloudFront. When a user requests your site, CloudFront silently slips this passcode into the envelope before passing it to the ALB. The ALB stands at the door, checks the envelope, and if the secret passcode is missing, it mercilessly drops the request into the void, returning a 403 Forbidden error. The user knows nothing. The malicious attacker trying to bypass CloudFront is shut out instantly. It is simple, deterministic, and devastatingly effective.

(Rea) 秘密の握手とは、普段は空っぽな貴様の脳味噌にしては驚くほど的を射た例えだな、イオち。そうだ。貴様は秘密のパスコード——例えば `X-My-Secret-Header: BloodOath123` のようなカスタムヘッダー——をCloudFrontに注入するのだ。ユーザーがサイトをリクエストすると、CloudFrontはALBに渡す前に、このパスコードを密かに封筒に滑り込ませる。ALBは扉の前に立ち、封筒を検め、もし秘密のパスコードが欠けていれば、無慈悲にもそのリクエストを虚無へと突き落とし、403 Forbiddenエラーを返すのだ。ユーザーは何も知る由はない。CloudFrontを迂回しようとする悪意ある攻撃者は即座に締め出される。単純で、決定的で、そして破壊的に効果的だ。

(Rindo) Exactly. We define a shared secret between CloudFront and the ALB.

1. In CloudFront, under Origin Settings, you add a Custom Header (e.g., Header Name: `X-Custom-Header`, Value: `RandomSecureString`).
2. In the ALB, you modify the Listener Rules. You set the default rule to return a fixed response (like a 403 error). Then, you create a new rule: *If* the HTTP header `X-Custom-Header` equals `RandomSecureString`, *then* forward the traffic to the EC2 target group.
This guarantees that only traffic flowing *through* CloudFront (and thus acquiring the secret header) can be processed by the ALB.

(Rindo) 全くその通りだ。CloudFrontとALBの間で共有のシークレットを定義するのだ。

1. CloudFrontのオリジン設定で、カスタムヘッダー（例：ヘッダー名 `X-Custom-Header`、値 `RandomSecureString`）を追加する。
2. ALBにて、リスナールールを変更する。デフォルトルールでは固定レスポンス（403エラーなど）を返すように設定する。そして新しいルールを作成する。*もし* HTTPヘッダー `X-Custom-Header` が `RandomSecureString` と等しい*ならば*、トラフィックをEC2ターゲットグループに転送する、という具合だ。
これにより、CloudFrontを*経由し*（結果としてシークレットヘッダーを取得した）トラフィックのみが、ALBによって処理されることが保証されるのだ。

(Io) Wow. Okay. That actually makes total perfect sense. CloudFront stamps the secret mark on the package, and ALB is the bouncer that only lets stamped packages in. (Rea) Hah. "Easy," she says. Do not grow complacent, child. While this satisfies the conditions of this rudimentary exam question, a true architect knows paranoia is a virtue. (Rea) ハッ。「余裕」だと。自惚れるなよ、小娘。この初歩的な設問の条件を満たしてはいるが、真のアーキテクトたる者、偏執狂的な疑念こそが美徳だと知っている。秘密の文字列が漏洩すれば、ヘッダーなど偽装可能だ。現実世界においては、このカスタムヘッダーの小細工と、先ほど我々が軽く切り捨てたマネージドプレフィックスリストを組み合わせ、偏執狂の独裁者が歓喜の涙を流すような多層防御アーキテクチャを構築するのだ。だが今日だけは、このささやかな小テストのためだけならば、貴様の「秘密の握手」への理解で十分だと言えよう。

(Rindo) Rea-san is correct. Security is never a single wall; it is a labyrinth. However, understanding the core mechanism of using custom HTTP headers for origin verification is a fundamental stepping stone. You have grasped the essence of the architecture, Iochi. The permanent frost in your mind has thawed just a fraction today. We shall continue to build your knowledge upon this foundation.

(Rindo) 怜空さんの言う通りだ。セキュリティは決して一枚の壁ではない。迷宮なのだ。しかし、オリジン検証のためにカスタムHTTPヘッダーを使用するという中核となるメカニズムを理解することは、不可欠な試金石である。お前はこのアーキテクチャの本質を掴んだぞ、イオち。お前の脳内の永久凍土が、今日ほんのわずかだが溶けたようだな。この基盤の上に、引き続きお前の知識を構築していくとしよう。
